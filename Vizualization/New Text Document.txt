You’re designing a **multi-state interactive report**, not just a simple dashboard. So the solution must be structured properly — otherwise bookmarks will start fighting each other.

Everything you described is solvable in Microsoft Power BI — but only if you separate responsibilities correctly.

---

# 🔷 First: Understand the Core Design Principle

You must separate:

1. **Layout control** (show/hide panels, resize visuals)
2. **View state control** (Dense / Longest / Gap)
3. **Data filtering** (Route slicer, table selection)

If you mix these in the same bookmark (especially with Data checked), things break.

---

# ✅ PART 1 — Pop-Up Filter Panel (Without Breaking Map State)

Your concern:

> If I open/close filter panel, will it reset my selected view (Dense / Longest)?

It WILL reset if you create the bookmark incorrectly.

### Correct Way To Build Filter Panel Bookmark

Create TWO bookmarks:

### 1️⃣ Filter_Open

* ✔ Display = ON
* ❌ Data = OFF
* Selected visuals = ON
* Only select:

  * Filter panel background
  * Slicer visuals inside panel

### 2️⃣ Filter_Close

* Same settings as above.

Important:

> Data must be OFF.

This ensures:

* Opening/closing filter panel does NOT touch Dense/Longest/Gap state.
* It only affects visibility.

---

# ✅ PART 2 — Three Buttons (Dense / Longest / Gap)

These are **View State Bookmarks**.

You need 3 bookmarks:

* Dense_View
* Longest_View
* Gap_View

For EACH of these:

### Bookmark Settings:

* ✔ Display = ON
* ❌ Data = OFF
* All visuals = ON

Why Data OFF?

Because these bookmarks are about:

* Showing correct table
* Resizing map
* Changing layout

NOT about changing route filters.

---

# 🔷 About Different Table Sizes & Map Resizing

Yes — you can resize the map per bookmark.

Bookmark will store:

* Position
* Size
* Visibility

So:

Dense → small table → large map
Longest → medium table → medium map
Gap → tall table → smaller map

Each bookmark stores its own layout.

That part is completely fine.

---

# ✅ PART 3 — The “Map Becomes Blank” Problem (Important)

You described:

> Route slicer has Route 110 selected
> Then I click a route in Top 10 table
> Map becomes blank

This is NOT a bookmark issue.

This is a filter conflict:

Route = 110
AND
Route = 220

= empty result.

### Solution (Professional Way)

Go to:
Format → Edit Interactions

Select the Top 10 table →
Turn OFF interaction with the Route slicer.

OR

Turn OFF slicer interaction with the Top 10 table.

You must prevent double filtering on the same column.

---

# ✅ PART 4 — How To Create a “Default” or “Blank” State

You absolutely should create a Default bookmark.

Call it:

Default_View

Set:

* All three Top 10 tables hidden
* Map full width
* No specific view active

Bookmark settings:

* ✔ Display ON
* ❌ Data OFF

Then create a button:
Action → Bookmark → Default_View

This becomes your reset layout button.

---

# 🔷 Very Important: DO NOT MIX Data + Display Unless Needed

Here’s the architecture you should follow:

| Bookmark Type                  | Data | Display | Purpose          |
| ------------------------------ | ---- | ------- | ---------------- |
| Filter Panel                   | OFF  | ON      | Show/Hide panel  |
| View State (Dense/Longest/Gap) | OFF  | ON      | Layout switching |
| Reset Filters (optional)       | ON   | ON      | Clear slicers    |

Never mix filter logic into layout bookmarks.

# ✅ OVERALL BUILD STRATEGY

You will build in this order:

1. **Finish all visuals first (no bookmarks yet)**
2. Create the 3 View bookmarks (Dense / Longest / Gap)
3. Create the Default layout bookmark
4. Build the Filter panel visuals
5. Create Filter Open/Close bookmarks
6. Fix visual interactions
7. Test navigation logic

Never create bookmarks while the layout is still changing.
